# review-app — Content Review & Audio-Splice tool

Internal web app for reviewing & correcting **staged VR-trip content (text + audio)** in
one GUI, replacing the manual "GDoc + audio-editor" relay. A reviewer opens a trip, edits
any field, regenerates/​splices the audio surgically, flags `done` / `edit required`,
comments, and autosaves; an admin reviews the diffs and submits to **Firebase staging**.
Stage 1 (now): admins use it themselves; **single combined role**, no login wall.

## Run (dev)
```bash
# backend — uses global py -3.12 (already has faster-whisper, firebase-admin, boto3)
cd backend && py -3.12 -m pip install -r requirements.txt
REVIEW_APP_TOKEN=dev-token py -3.12 -m uvicorn --app-dir . app.main:app --host 127.0.0.1 --port 8000
# frontend
cd frontend && npm install && npm run dev      # http://127.0.0.1:5173  (the app)
```
Open **5173** (the UI). `:8000` is the JSON API — opening it directly looks "broken". Both
must run. Token: `X-Review-Token` (FE env `VITE_REVIEW_TOKEN`, BE env `REVIEW_APP_TOKEN`,
default `dev-token`). uvicorn binds 127.0.0.1 only; the FE Vite proxy forwards `/api`,
`/audio`, `/overlays` → `:8000`.

## Architecture
- **frontend/** React 19 + Vite 7 + Tailwind (mirrors `..\library-app`). Talks ONLY to the
  backend via `src/api.ts` (no direct Firestore). Scene media = the **thumbnail JPG**
  (`thumb_url`), not a Vimeo player.
- **backend/** FastAPI (Python 3.12). **Review state in SQLite** `backend/review.db`
  (autosaves every edit/flag/comment/coverage). Per-session audio under
  `backend/work/{sid}/` (orig=pristine v0, working=current, versions, candidate, fallback).
  **Masters are never mutated during review**; submit promotes the corrected working files.
- **Source of truth = the staging Firebase `Trips/{contentId}` doc** (+ the `TripGroups`
  doc for the description/categories). The GDoc is out of the loop.

## Audio (MP3 end-to-end)
- **Sources** (`sessions.resolve_audio_dir`): Quicktrips masters
  (`stage9.common.paths_for`) → `Audio Generation/<trip>/` (England A12/B1) →
  **`Audio Generation/Sent to KP/MP3/<trip>/`** (Japan `_EN`) → shallow nested search.
- **Level speed** (`audio_core.speed_for_trip`, honoured by `eleven_multilingual_v2`):
  **A12=0.7, B1=0.85, B2+/native=1.0**. Applied at seed/regenerate/fallback. *Japanese uses
  the v3 API where speed is always 1.0; HSK Mandarin is TBD* — branch by language/model
  there when those get backend support.
- **Splice engine** `audio_splice.py` (SceneDesc only; Q&A + "whole" = full regen). The
  hard-won correctness points (DON'T regress — two red-teams): cut times from **raw Whisper
  `word.start/end`** via SequenceMatcher (NOT `subtitles.token_timeline`); **anchor-verify
  → `edit_required`** on mismatch; **non-Latin / Gemini-fallback → `edit_required`**;
  **pause-aligned anchors + seam gate**; **level-match the candidate to the kept context
  before the cut**; **boundary cuts are NOT energy-refined** (they'd truncate the new
  audio); splice in **mp3-PCM, peak-limit only the insert**; the human listen is
  load-bearing. `done` only unlocks after ≥95% genuine playback (seek-proof) and resets when
  the audio/text changes.
- **Versioning:** canonical `<i>.mp3`; superseded takes archived `versions/<i>v<n>.mp3`.
- **Submit** writes changed **text** to staging Trip + TripGroup (desc + re-derived
  `tripCategories`) and leaves the corrected `<i>.mp3` masters in place — **Stage 9**
  (`stage9_finalise.py` / the CEFR `run_levels.py`) does the mp3→ogg + subtitles + S3.

## Cloudflare R2
- **Thumbnails:** `thumbs.py` maps scene `videoUrl` → `VRD\VideoIds-…json` (stem from the
  entry's `url`) → regional `…VID-PIC Thumbnails\**\Vid ….jpg` → bucket
  `dynamic-languages-thumbs` key `scene-thumbs/<stem>.jpg`, served at
  `https://thumbs.dynamiclanguages.org/`.
- **Review audio:** `review_audio.py` (+ `Scripts\upload_review_audio_r2.py` for bulk, +
  the stage-5c hook in `run_levels.py`) push mp3s to bucket **`review-audio`** key
  `<contentID>/<file>.mp3`, served at **`https://reviewaudio.dynamiclanguages.org/`**. On
  combine/import/fallback the review app uploads the corrected take + `<i>v<n>.mp3` +
  `<i>_fallback.mp3`. Uploads are **best-effort** (never break a review op). Creds =
  `Cloudfare_*` in the Scripts `.env`. The backend needs **no AWS write** (S3 is Stage 9's).

## Remote review / deployment
For letting a reviewer work **off-machine**. The audio *bytes* are public at
`reviewaudio.dynamiclanguages.org`, but that is **NOT enough** — see below.

**Not public today:** the frontend (`:5173`) and the backend (`:8000`, bound to 127.0.0.1).
And the running app still *serves* audio through the **local backend `/audio`** (from
`backend/work/`) — only **thumbnails** load from R2 today.

**Why audio-on-R2 isn't enough:** the real review (edit text → regenerate/​splice → combine →
submit) runs the **splice engine on the backend** (ElevenLabs + Whisper + ffmpeg + the master
audio). So the **backend must be reachable** — public audio removes only one blocker.

**Path A — tunnel (near-term, ~hours):** run `cloudflared`/`ngrok` over the locally-running
app; the reviewer gets a URL, the backend stays here with all its files/GPU. Requires only:
(1) a **strong `REVIEW_APP_TOKEN`** (not `dev-token` — an open tunnel exposes `/regenerate`
[credit burn] and `/submit` [staging write]); (2) this machine stays on, both servers up.

**Path B — deploy (proper, the real lift):**
1. **Frontend** → Vercel (mirrors library-app). Easy.
2. **Backend** → a host — the substantive work; it depends on the **Scripts modules**,
   **ffmpeg**, a **GPU** for Whisper (or slow CPU), the **Firebase key + EL/Gemini/R2
   secrets**, and the **source audio**. *Audio-on-R2 helps:* change the backend to **fetch
   originals from `review-audio`** instead of needing local masters — removes the biggest
   local-file dependency.
3. **Real auth** — the reviewer/admin login (stage-2 role split); a public backend can't run
   on a shared token.
4. *(Optional)* point the player at the **R2 audio domain** to offload audio serving — but
   in-progress candidates (pre-combine) still come from the backend.
